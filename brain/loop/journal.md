## 2026-07-15 · Iteration 9 — PH payer rulebook + policy intelligence layer
Goal (/goal): deep-research the actual PH denial/eligibility clocks (PhilHealth + HMO, group vs individual policies), validate the assumptions the product was running on, and implement the result as a real intelligence layer across both agents. Research first, code second.

- **Research (3 parallel primary-source passes)** — landed in [[ph-denial-and-eligibility-rules]]. The folklore was wrong in both directions: PhilHealth's MR window is **15 calendar days** (PC 03 s.2008 — not the widely-repeated 60), the 60-day filing rule is **statutory** (RA 7875 §35 as amended, not Circular 2018-0014), RTH refile is 60d from notice, and "LOA valid 30 days" is **Maxicare-specific** (PhilCare's official window is 3 calendar days; MediCard/Intellicare publish none). No IC circular fixes an HMO claims deadline — IMC 2023-01 CAMS (7/45 wd) + payer-set windows (MediCard: appeal within 10 wd) govern; IC CL 2018-65 caps PEC waiting at 1 year (the regulatory anchor for the fixture terms). Also mapped: group-vs-individual underwriting/PEC/MBL/grace differences and the documented point-of-service "not eligible" taxonomy.
- **Knowledge module** (`@helix/payers/knowledge/phRules.ts`, ADR-011) — every window rule carries authority + document ref + confidence (`verified`/`reported`/`assumed`) + `verifyBeforeLive`; contractual HMO defaults are conservative (filing 30d, reconsideration 30d ceiling, LOA 3d) and hard-flagged for per-contract confirmation. Pure date math (`assessDeadline` → deadline/daysRemaining/urgency) with UTC day precision.
- **Policy engine** (`services/agents/src/policy/engine.ts`) — new adapter capability `getPolicyProfile` (fixture-driven member+plan join) feeds deterministic, cited `PolicyCheck`s: coverage window, waiting period (individual plans), PEC (attention-only — whether a condition is pre-existing is clinical, so Helix flags the RULE and cites the IC ceiling, never judges), benefit limit (MBL exhausted/≥80%), forward-looking claim-filing window. One-way escalation: fail ⇒ ineligible, attention/unknown ⇒ needs_review, never downgrades the adapter or confidence gate. Failures become blocking gaps; HMO LOA drafts now carry their validity note.
- **Revenue agent upgraded** — the flat 30/60 age gates are gone; recoverability now derives from the payer-kind rulebook window (PhilHealth 15d MR / 60d RTH refile for correctable claims; HMO 30d default), every governed finding carries a `DeadlineAssessment` (basis/deadline/days-left/urgency), and rationales cite the rule (`reg:philhealth/philhealth-appeal`, …). service_not_covered gets the window as guidance without gating; HMO document fixes stay clock-independent (no published window — silence never pretends to be a rule).
- **Surfaces** — /verify decision card gains a "Policy checks" section (status-accented rows, cited, with the note that checks are deterministic); /revenue gains a "Recovery window" column (days-left countdown → date, "Window closed", or explicit "Not time-bound"). All new strings EN+FIL under the compile-enforced parity. Fixtures now model the demo arc: clean corporate member, individual member mid-waiting-period (hard block), individual inside the 12-month PEC window (needs review), corporate member with exhausted MBL (hard block) — the Sample button loads the clean member so the happy path stays the demo default.
- **Persistence** — `eligibility_checks.policy_checks` jsonb (additive, nullable, rollback noted; applied to live Supabase via direct ALTER — drizzle journal still predates the pushed schema) and proposal reconstruction carries it back.
